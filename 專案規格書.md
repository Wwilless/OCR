**版本：** 2.0.0
**日期：** 2026-05-20
**作者：** willes.chen

---

## 1. 專案概述

本系統透過工業攝影機自動偵測培養瓶是否進入視野，並即時辨識瓶身上的批號資訊（MFG / MFD、LOT），以 REST API 供外部系統整合。

---

## 2. 系統架構

```
┌─────────────────────────────────────────┐
│           外部系統 / Postman             │
│  POST http://<IP>:8000/scan             │
│  Header: X-Api-Key: OCR-2026-KEY-...    │
└──────────────────┬──────────────────────┘
                   │ HTTP
┌──────────────────▼──────────────────────┐
│           fastapi_server.exe            │
│           Port: 8000（自動換 port）      │
│  ┌──────────────────────────────────┐   │
│  │  capture_ocr（同 process import） │   │
│  │  ┌────────┐ ┌──────┐ ┌────────┐ │   │
│  │  │EasyOCR │ │TrOCR │ │Paddle  │ │   │
│  │  └────────┘ └──────┘ └────────┘ │   │
│  └──────────────────────────────────┘   │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────▼──────────────────────┐
│        攝影機（索引設定於 config.json）   │
│        解析度：645 × 560 px              │

```

> **注意：** capture_ocr 直接以 Python import 方式整合進 fastapi_server，非獨立子行程，模型僅於伺服器啟動時預載一次。

## 3. 元件說明

### 3.1 OCR 引擎（`capture_ocr.py`，整合於 `fastapi_server.exe`）

#### 攝影機設定

| 參數 | 值 | 說明 |
|------|----|------|
| 裝置編號 | 由 `config.json` 讀取，預設 `1` | 可透過 patch.exe 修改 |
| 解析度 | 645 × 560 px | 固定值 |

#### ROI 偵測區域（格式：x, y, w, h）

| 名稱 | 座標 | 用途 |
|------|------|------|
| 綠框 ROI | (97, 67, 424, 371) | OCR 辨識範圍 |
| 藍框 BLUE_ROI | (84, 13, 427, 50) | 瓶口方向偵測 |
| 紅框 RED_ROI | (526, 8, 99, 102) | 有無瓶子偵測 |

#### 偵測邏輯

```
每 2 秒偵測紅框暗色占比
  ├─ 占比 < 20%  → 有瓶子
  │   ├─ 藍框暗色占比 >= 50%  → 方向正確 → 觸發 OCR
  │   └─ 藍框暗色占比 < 50%   → 輸出「培養瓶裝反了」
  └─ 占比 >= 20% → 無瓶子 → 輸出「無瓶子」
```

#### OCR 三引擎流程

```
Step 1：EasyOCR   → 偵測文字位置（Bounding Box）
Step 2：TrOCR     → 依框位逐塊精細辨識（與 Step 3 並行）
Step 3：PaddleOCR → 依框位逐塊辨識（與 Step 2 並行）
Step 4：三引擎結果逐字元多數決投票
Step 5：套用領域規則（前綴修正、MFG/LOT 格式驗證）
```

#### 輸出標籤格式（固定 2 行）

| 標籤 | 格式 | 說明 |
|------|------|------|
| MFG 或 MFD | 8 位純數字 | 第 1 行，由 OCR 從影像辨識決定 |
| LOT | 2 位數字 + 1 位月份字母(A-L) + 3 位數字 | 第 2 行，固定標籤 |

## 4. API 規格

### `GET /health`

確認伺服器與模型載入狀態。

**Request Header：**
```
X-Api-Key: OCR-2026-KEY-a3f7d2b9e1c458f0
```

**回傳範例（模型載入中）：**
```json
{
  "status": 1,
  "models_ready": false,
  "models_error": ""
}
```

**回傳範例（就緒）：**
```json
{
  "status": 1,
  "models_ready": true,
  "models_error": ""
}
```

---

### `POST /scan`

觸發單次 OCR 辨識。

**Request：** 無需 Body

**Request Header：**
```
X-Api-Key: OCR-2026-KEY-a3f7d2b9e1c458f0
```

**Response：**

| status 值 | 情況 | HTTP Code |
|-----------|------|-----------|
| `1` | 辨識成功 | 200 |
| `-1` | 無瓶子 | 200 |
| `0` | 培養瓶裝反了 | 200 |
| `-2` | 系統錯誤 / 模型載入中 | 500 / 503 |

**成功範例（MFG）：**
```json
{
  "status": 1,
  "results": {
    "MFG": "20240115",
    "LOT": "24A001"
  }
}
```

**成功範例（MFD）：**
```json
{
  "status": 1,
  "results": {
    "MFD": "20250115",
    "LOT": "24A001"
  }
}
```

**無瓶子：**
```json
{ "status": -1, "message": "無瓶子" }
```

**方向錯誤：**
```json
{ "status": 0, "message": "培養瓶裝反了" }
```

**錯誤：**
```json
{ "status": -2, "message": "錯誤描述" }
```

## 5. 技術依賴

### OCR 引擎套件

| 套件 | 版本 | 用途 |
|------|------|------|
| easyocr | ≥1.7 | 文字位置偵測 |
| transformers | ≥4.30 | TrOCR 模型（microsoft/trocr-small-printed） |
| paddleocr | ≥2.7 | PaddleOCR 英文引擎 |
| paddlepaddle | ≥2.6 | Paddle 底層框架 |
| opencv-python | ≥4.8 | 攝影機讀取與影像前處理 |
| torch | CPU-only | TrOCR 推理（不需 GPU） |
| numpy | ≥1.23 | 數值運算 |
| pillow | ≥10.0 | 影像格式轉換 |

### API 伺服器套件

| 套件 | 版本 | 用途 |
|------|------|------|
| fastapi | ≥0.136 | REST API 框架 |
| uvicorn | ≥0.47 | ASGI 伺服器 |

### 打包工具

| 套件 | 版本 | 用途 |
|------|------|------|
| pyinstaller | ≥6.20 | 打包為獨立 exe |

## 6. 打包與部署

### 部署資料夾結構

```
OCR_api\
  ├── patch.exe            ← 第一次使用時執行（設定 + 下載模型）
  ├── fastapi_server.exe   ← 每次啟動伺服器執行
  └── _internal\           ← 套件資料夾（勿刪除）
```

自動產生的設定與模型檔案：

| 檔案 / 資料夾 | 位置 | 說明 |
|--------------|------|------|
| `config.json` | `OCR_api\` 同層 | 攝影機索引、Port 設定 |
| EasyOCR 模型 | `%USERPROFILE%\.EasyOCR\` | 約 50 MB |
| TrOCR 模型 | `%USERPROFILE%\.cache\huggingface\` | 約 300 MB |
| PaddleOCR 模型 | `%USERPROFILE%\.paddleocr\` | 約 200 MB |

### 打包指令（開發端）

```powershell
# 在 d:\OCR 目錄下執行
build.bat
# 輸出位置：dist\OCR_api\
```

### 首次使用步驟

```
1. 接上攝影機（USB）
2. 執行 patch.exe
   → 輸入攝影機索引（預設 1，直接 Enter 略過）
   → 自動下載 OCR 模型（約 550 MB，需網路）
3. 執行 fastapi_server.exe
   → 等待顯示「OCR 模型預載完成」
4. 外部系統呼叫 POST http://<SERVER_IP>:8000/scan
```

### 之後每次使用

直接執行 `fastapi_server.exe`，不需再執行 patch.exe。

### 驗證攝影機設定

```
執行 fastapi_server.exe 後，呼叫 POST /scan：
  → 回傳 OCR 結果 或 {"status": -1} → 攝影機索引正確
  → 回傳 {"status": -2, "message": "找不到攝影機"} → 重新執行 patch.exe 修改索引
```

## 7. 已知限制

| 限制 | 說明 |
|------|------|
| Windows 限定 | 使用 `cv2.CAP_DSHOW`，僅支援 Windows 平台 |
| 模型首次下載 | 需網路連線，約 550 MB，由 patch.exe 處理 |
| 防毒軟體誤判 | PyInstaller exe 可能被標記，需手動加入 Windows Defender 排除清單 |
| PaddleOCR 失敗備案 | 若 PaddleOCR 載入失敗，伺服器以 EasyOCR + TrOCR 雙引擎繼續運作 |
